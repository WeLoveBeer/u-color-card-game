# U彩牌 UI 设计文档

版本：1.0  
平台：微信小游戏  
首版范围：人机对战 + 好友房  
设计目标：轻量、清楚、快速开局，突出“纯随机发牌，不控牌局”

## 1. 整体视觉方向

关键词：

- 干净
- 明快
- 公平
- 轻竞技
- 微信小游戏友好

主色建议：

- 背景：深蓝或浅灰蓝，避免过度花哨。
- 主按钮：绿色或蓝色，突出微信环境和行动感。
- 强调色：红、黄、蓝、绿，对应牌面颜色。
- 公平标识：白色或金色小徽章。

风格：

- 卡牌边角可圆润，但整体不要过度卡通。
- 主界面信息密度适中。
- 对局界面优先保证手牌、当前颜色、轮到谁清晰可见。

## 7. 对局界面

### 7.1 竖屏整体布局

顶部：

- 对手头像区。
- 每个对手显示剩余手牌数量。
- 当前回合玩家高亮。

中部：

- 当前颜色指示。
- 出牌方向箭头。
- 摸牌堆，作为摸牌交互入口。
- 弃牌堆。
- 最近动作提示。
- 功能牌动画层。

底部：

- 玩家头像和剩余倒计时。
- 玩家手牌区。

角落：

- 左上：退出或托管。
- 右上：设置、规则。

### 7.2 玩家手牌

展示方式：

- 手牌横向扇形或平铺。
- 可出的牌高亮。
- 不可出的牌降低透明度。
- 选中的牌上浮。
- 轮到自己时，所有可出的牌自动上提一小段距离。

操作：

- 点击可出牌先选中，卡片上移。
- 再点击“出牌”按钮确认打出。
- 双击可出牌可作为快捷出牌，首版可选做。
- 点击不可出牌弹出轻提示。
- 点击已选中的牌或点击空白区域取消选中。
- 手牌过多时支持横向滑动。

手牌层级：

- 选中的牌显示在最上层，避免被相邻卡遮挡。
- 可出牌自动上提后仍保持原始排序。
- 自动上提不改变实际手牌顺序。

上提距离建议：

- 可出牌自动上提：16px 到 24px。
- 用户点击选中：40px 到 56px。
- 不可出牌点击反馈：轻微左右摇动 120ms。

### 7.3 出牌操作区

出牌操作区位于手牌上方或右下角，只有轮到自己时显示。

按钮：

- 出牌。

说明：

- 对局界面只保留一个明确按钮：“出牌”。
- 不显示“提示”按钮，因为轮到自己时系统会自动把可出牌上提，已经起到提示作用。
- 不显示“摸牌”按钮，摸牌交互放到摸牌堆上，通过发光箭头引导。

按钮状态：

- 未选中牌时，“出牌”按钮置灰。
- 选中可出牌时，“出牌”按钮高亮。
- 没有选牌或不在自己回合时，出牌按钮不显示或置灰。
- 没有可出牌时，出牌按钮置灰，摸牌堆出现发光箭头。

出牌按钮逻辑：

- 玩家选中一张可出牌后点击“出牌”。
- 如果选中的是变色牌或强制摸四牌，先弹出颜色选择，再提交出牌。
- 提交后等待服务端确认。
- 服务端确认前显示按钮加载态，禁止重复点击。
- 服务端拒绝时，卡牌回到原位并显示错误提示。

自动上提逻辑：

- 回合切到自己时，客户端根据当前服务端状态计算可出牌列表。
- 所有可出牌自动上提 16px 到 24px，并增加高亮边框。
- 如果只有一张可出牌，可轻微呼吸高亮。
- 如果 3 秒内玩家无操作，可出牌轻微呼吸高亮一次。
- 回合结束或玩家摸牌后，重新计算可出牌列表。

### 7.4 摸牌区

摸牌堆显示在中部偏左或牌桌中央靠左位置。界面只保留一个摸牌堆，不在右侧额外再放一堆牌，避免误导玩家。

状态：

- 玩家无牌可出时，摸牌堆高亮。
- 玩家可出牌时，摸牌堆正常显示。
- 玩家可出牌时，首版默认不允许主动摸牌，减少规则争议。

摸牌箭头：

- 当玩家无牌可出时，摸牌堆上方显示一个向下或指向牌堆的小箭头。
- 箭头使用亮色描边和柔和发光。
- 箭头持续上下跳动，提示玩家点击牌堆摸牌。
- 玩家点击箭头或摸牌堆，触发摸牌请求。
- 服务端确认后播放摸牌动画。
- 摸牌完成后，箭头隐藏。

### 7.5 弃牌区

弃牌堆显示当前顶部牌。

需要展示：

- 当前牌面。
- 当前有效颜色。
- 功能牌效果动画。

变色后：

- 弃牌堆周围显示当前指定颜色的光圈。

不显示内容：

- 弃牌区不额外显示“转”字标签。
- 弃牌区下方不显示红黄蓝绿四色条。
- 当前颜色只通过弃牌堆外圈、顶部小色点或当前颜色指示器表达。

### 7.6 中央方向箭头

牌桌中央需要常驻一个出牌方向箭头。

默认状态：

- 顺时针时显示顺时针圆形箭头。
- 逆时针时显示逆时针圆形箭头。
- 箭头使用低透明度，不遮挡牌堆。

Reverse 触发时：

- 中央箭头放大到 1.4 倍。
- 箭头沿新方向快速旋转一圈。
- 屏幕中部显示“方向反转”短文案。
- 对手座位的高亮顺序按新方向依次闪烁。
- 动画结束后，中央箭头保持新方向。
- 不在牌面或桌面额外显示“转”字，避免和 Reverse 箭头重复。

### 7.7 回合提示

轮到自己：

```text
轮到你出牌
```

轮到他人：

```text
等待对手出牌
```

功能牌提示：

```text
你被禁行，跳过一回合
你需要摸 2 张
出牌方向已反转
```

### 7.8 倒计时

倒计时显示在当前玩家头像旁。

状态：

- 30 秒到 10 秒：正常颜色。
- 10 秒以内：黄色提醒。
- 5 秒以内：红色闪烁。

超时处理：

- 服务端自动托管。
- 有可出牌则自动出一张。
- 无可出牌则自动摸牌。

## 8. 对局动画设计

### 8.1 打牌动画

触发：

- 玩家或 AI 成功打出一张牌。

动画流程：

1. 出牌者手牌中的目标牌上浮。
2. 牌从手牌区飞向弃牌堆。
3. 飞行过程中轻微缩放，落到弃牌堆时弹一下。
4. 弃牌堆顶部牌更新。
5. 如果牌改变当前颜色，弃牌堆外圈切换到新颜色。

时长建议：

- 普通打牌：250ms 到 350ms。
- AI 打牌：可略快，200ms 到 300ms。

表现要求：

- 动画期间锁定重复点击。
- 牌面必须清楚可见。
- 动画结束后再切换回合高亮。

### 8.2 摸牌动画

触发：

- 玩家点击摸牌堆上的发光箭头或摸牌堆。
- 被 +2 或 +4 影响强制摸牌。
- AI 摸牌。

普通摸牌流程：

1. 摸牌堆顶部出现一张牌背。
2. 牌背从摸牌堆飞到目标玩家手牌区。
3. 如果是自己摸牌，落入手牌后翻面显示。
4. 如果是其他玩家摸牌，只增加手牌数量，不展示具体牌面。

强制摸牌流程：

1. 目标玩家头像旁显示“摸 2 张”或“摸 4 张”。
2. 多张牌背从摸牌堆依次飞向目标玩家。
3. 每张牌间隔 80ms 到 120ms。
4. 摸牌结束后显示“跳过回合”提示。

表现要求：

- 多张摸牌不应同时堆叠到看不清。
- 其他玩家摸牌不能泄露牌面。
- 摸牌后手牌数量实时更新。

### 8.3 Skip 动画

触发：

- 打出禁行牌 / Skip 类牌。

动画流程：

1. 禁行牌落到弃牌堆后，牌面发出短促闪光。
2. 下家头像上方出现“跳过”标记。
3. 下家头像变灰 0.5 秒。
4. 一条禁止符号从弃牌堆飞向下家头像。
5. 回合高亮直接跳到下一位有效玩家。

时长建议：

- 总时长 700ms 到 900ms。

文案：

```text
跳过一回合
```

### 8.4 +2 动画

触发：

- 打出加二牌。

动画流程：

1. +2 牌落到弃牌堆后，牌面数字 +2 放大闪烁。
2. 目标玩家头像旁显示“+2”。
3. 两张牌背从摸牌堆依次飞向目标玩家。
4. 目标玩家头像短暂震动。
5. 显示“摸 2 张，跳过回合”。
6. 回合进入下一位有效玩家。

如果开启加二叠加：

- 目标玩家可在倒计时内继续打出 +2。
- 叠加时中央显示累计数量，例如“+4”“+6”。
- 最终无法叠加的玩家按累计数量摸牌。

首版默认：

- 加二叠加关闭。

### 8.5 Reverse 动画

触发：

- 打出转向牌 / Reverse 类牌。

动画流程：

1. Reverse 牌落到弃牌堆。
2. 牌桌中央方向箭头从低透明度变为高亮。
3. 箭头按新方向旋转一圈。
4. 箭头颜色短暂变为当前牌颜色。
5. 所有玩家座位边框按新出牌方向依次亮起。
6. 中央显示“方向反转”。
7. 动画结束后，回合切到新方向上的下一位玩家。

2 人局特殊处理：

- Reverse 等同于 Skip。
- 仍播放 Reverse 箭头动画，但同时显示“对手跳过”。

时长建议：

- 900ms 到 1100ms。

### 8.6 变色动画

触发：

- 打出变色牌或强制摸四牌。

动画流程：

1. 牌落到弃牌堆。
2. 弹出颜色选择面板。
3. 玩家选择颜色后，四种颜色粒子向选中颜色收束。
4. 弃牌堆外圈切换到选中颜色。
5. 当前颜色指示器同步更新。

AI 选择颜色：

- 不弹出选择面板。
- 直接播放颜色切换动画。

### 8.7 动画优先级

如果同一张牌同时触发多个效果，播放顺序为：

```text
打牌飞行动画
功能牌主动画
摸牌或跳过动画
回合切换动画
```

动画播放期间：

- 禁止重复出牌。
- 禁止重复摸牌。
- 允许打开设置和规则，但不暂停服务端倒计时。

### 13.2 结算广告奖励

人机结算页：

```text
观看视频，金币奖励翻倍
```

好友房：

## 14. 空状态与错误状态

### 14.1 网络断开

提示：

```text
网络连接断开，正在重连...
```

重连成功：

```text
已恢复对局
```

重连失败：

```text
重连失败，本局将由 AI 托管
```

### 14.2 房间不存在

提示：

```text
房间不存在或已结束
```

按钮：

- 重新输入。
- 返回大厅。

### 14.3 非法操作

提示：

```text
这张牌现在不能出
```

或：

```text
还没轮到你
```

## 16. UI 验收标准

- 首页 3 秒内能看懂主入口。
- 玩家 2 次点击内能进入人机对战。
- 好友房创建流程不超过 4 步。
- 对局中可出牌和不可出牌状态明显。
- 当前颜色始终清楚可见。
- 2 人、3 人、4 人局的对手区都不拥挤。
- 手牌超过 12 张时仍可操作。
- 断线、超时、非法操作都有明确反馈。
- 广告入口不干扰核心对局。
